# Context tóm tắt cho chat mới

## Mục tiêu chính của project / bug
- Project `BaccaratSexyCasino` là app desktop tự động theo dõi dữ liệu game live (đặc biệt Baccarat/Xóc Đĩa), đọc trạng thái ván từ trang game qua WebView2 + JS bridge, rồi chạy chiến lược cược.
- Bug chính hiện tại: chạy `new.wencheng.cc` thì đọc dữ liệu game OK, nhưng chạy `vipbet389.com` thì không bắt được dữ liệu game.
- Trọng tâm bug: `vipbet389.com` bọc game trong iframe cross-origin (`gci.usplaynet.com`), luồng vào game không chuyển sang URL game thật (`singleBacTable/webMain/gamehall`) nên bridge không có tick dữ liệu.

## Tech stack (ngôn ngữ, framework, DB…)
- Ngôn ngữ chính: `C#` + `JavaScript`.
- UI/Desktop: `WPF` (.NET, project đang build theo `net8.0-windows`).
- Browser engine: `Microsoft.Web.WebView2` (main web + popup web + frame hook).
- Data local: file JSON (`config.json`, `stats.json`) và log text theo ngày.
- Không thấy DB server trong scope bug hiện tại.
- Có tích hợp `ABX.Core` (dependency nội bộ, hiện build local đang thiếu dll ở vài lần build test).

## Những lỗi đã gặp & nguyên nhân chính
- Lỗi kỹ thuật quan sát được từ log:
  - `Blocked a frame with origin "https://vipbet389.com" from accessing a cross-origin frame.`
  - Khi route popup trực tiếp vào `apiLogin` từng gặp:
    - `status=1008`
    - `WalletLiveToken is not match`
- Tình trạng mới nhất (sau các patch):
  - `HTTP.resp/player-flow` chỉ thấy `apiLogin ... status=200`.
  - `HostLaunchProbe` cho thấy iframe luôn là `apiLogin`, không đổi sang `singleBacTable/webMain/gamehall`.
  - `WaitForBetGameUrl` timeout nhiều lần.
  - `BridgeProbe` tại top luôn `read/start/bet = undefined`.
- Nguyên nhân chính hiện tại:
  - Click hiện tại khả năng bấm nhầm target UI (promo/card), không kích hoạt đúng flow mở game table.
  - Hoặc flow mở game thực sự chạy trong iframe cross-origin mà app chưa inject được vào context frame đang hoạt động.
  - Popup-route bằng `apiLogin` không bền vì token/session có thể ràng buộc theo flow gốc (gây mismatch).

## Các giải pháp đã thử, giải pháp hiện tại đang dùng
- Đã thử và giữ:
  - Mở rộng nhận diện trang game trong JS (`singleBacTable/gamehall`).
  - Bổ sung bridge probe toàn diện (`[BridgeProbe]`) ở main/popup/frame.
  - Bổ sung probe host theo thời gian (`[HostLaunchProbe]`) trước/sau click + trace 12s.
  - Bổ sung log network `player-flow` để bắt URL `/player/*`.
  - Parse và log provider error (`[PopupWeb][ProviderError]`, `[VaoXocDia][provider-error]`).
  - Nới gate pipeline bằng tín hiệu tick thay vì phụ thuộc tuyệt đối vào `Web.Source`.
- Đã thử nhưng không giữ làm đường chính:
  - Route popup sang iframe `apiLogin` để lấy context game.
  - Kết quả từng rơi vào `WalletLiveToken is not match`, nên hiện có chặn route popup nếu candidate chỉ là `apiLogin`.
- Giải pháp hiện tại trong code:
  - Ưu tiên giữ trên host, click mở game.
  - Nếu chưa có game signal thì force reload iframe candidate để kích lại frame lifecycle/injection.
  - Theo dõi thêm bằng `TickDiag`, `WaitForGameSignal`, `HostLaunchProbe`, `player-flow`.
