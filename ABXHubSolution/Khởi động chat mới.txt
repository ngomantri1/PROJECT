# Context Tóm Tắt Cho Chat Mới

## Mục tiêu chính của project / bug
- Project: công cụ desktop WPF để bọc WebView2, vào game Baccarat (VivoGaming), đọc trạng thái game realtime, đồng bộ dữ liệu sang bảng điều khiển để auto-bet.
- Bug gốc: từ site cũ (Sexy) chuyển sang Vivo thì `Canvas Watch` không hiện hoặc hiện thiếu dữ liệu.
- Bug hiện tại cần xử lý tiếp: khi load vào game app bị đơ, bảng điều khiển bị lag/không thao tác ổn, có lúc không bấm được `Đăng nhập Tool`.
- Yêu cầu nghiệp vụ đã chốt:
1. `Đăng nhập Tool` chỉ check bản quyền/tài khoản hoạt động.
2. Không gắn luồng detect need-login/autofill/click login như trước.
3. Countdown đồng bộ theo `% Prog` (không theo giây).
4. Hỗ trợ URL Vivo (`games.vivogaming.com`, `selectedgame=baccarat`), bỏ ràng buộc `singleBac only`.

## Tech stack (ngôn ngữ, framework, DB…)
- Ngôn ngữ: C#, JavaScript.
- UI/Desktop: WPF (.NET 8, `net8.0-windows`).
- Web embed: `Microsoft.Web.WebView2`.
- Kiến trúc: desktop app + JS bridge inject vào WebView2/frame.
- Dữ liệu lưu local: JSON config/stats trong `%LOCALAPPDATA%\BaccaratViVoGaming`.
- Log local: `%LOCALAPPDATA%\BaccaratViVoGaming\logs\yyyyMMdd.log`.
- Bản quyền: fetch JSON từ GitHub raw (không DB server nội bộ).
- Chế độ build:
1. Debug dùng plugin mode cho AutoBetHub/ABX.Core.
2. Release self-contained single-file.

## Logic chính đã thiết kế (tên hàm, class, luồng xử lý)
- `VaoXocDia_Click(...)`
- Vai trò hiện tại: chỉ xác thực bản quyền (`EnsureLicenseAsync`) + đảm bảo bridge.
- Không còn check game-ready ở nút login tool.

- `PlayXocDia_Click(...)`
- Vai trò: bắt đầu cược, có check context game bằng `EnsureGameContextReadyForPlayAsync(...)`.

- `EnsureGameContextReadyForPlayAsync(...)`
- Chuỗi fallback khi chưa thấy game:
1. check tín hiệu game gần nhất
2. wait URL game (`WaitForBetGameUrlAsync`)
3. wait tick game (`WaitForGameSignalAsync`)
4. route/reload iframe/popup fallback
5. log probe chẩn đoán nếu fail

- `HandleIncomingWebMessageAsync(string msg, string source)`
- Nhận message bridge (`tick`, `cwLogBatch`, `result`, ...).
- Parse snapshot bằng `ParseCwSnapshotLoose`.
- Gộp nguồn tick từ `main-msg`, `main-frame`, `popup-frame`, `popup-pull`.
- Cập nhật UI progress/status/kết quả/username/số dư.

- `TryPullPopupTickFallbackAsync()`
- Poll fallback từ popup khi không có tick push.
- Execute JS `PULL_POPUP_TICK_NOW`.

- `NormalizeProgressPercent(double? prog, string? source)`
- Chuẩn hóa `prog` về %.
- Nhận diện source ratio/seconds để map đúng.

- `BuildStatusUiText(statusRaw, prog)`
- Map status DOM thành text UI.

- `SetModeUi(bool isGame)` + `ApplyUiMode(bool isGame)`
- Điều khiển ẩn/hiện nhóm login, strategy, status, stats.

- `ApplyMouseShieldFromCheck()`
- Điều khiển khóa chuột WebView theo checkbox/tab running.

## Những lỗi đã gặp & nguyên nhân chính
- Canvas Watch không load lúc chuyển sang Vivo.
- Nguyên nhân: điều kiện JS/URL cũ ưu tiên `singleBacTable.jsp` + chưa match đủ Vivo host/query.

- Canvas Watch hiện nhưng thiếu dữ liệu (tên nhân vật, tài khoản, trạng thái).
- Nguyên nhân: tail selector cũ không còn đúng với DOM Vivo.

- `TextMap` lúc đầu không hoạt động đúng ở trang mới.
- Nguyên nhân: context/frame/tail cũ, lấy nhầm trang lobby hoặc link cũ.

- Nút `Đăng nhập Tool` từng không bấm được hoặc bị chặn logic.
- Nguyên nhân: trước đó có check game context/ready không phù hợp với nghiệp vụ login-only.

- Countdown/prog không đẩy JS -> C# hoặc hiển thị sai.
- Nguyên nhân:
1. Chuyển đổi seconds/ratio/% chưa đồng nhất.
2. Tick nguồn yếu (`popup-pull`) có `prog=-` ghi đè.
3. UI hiển thị từng bị neo theo giây cũ.

- Thời gian trạng thái từng bị “kẹt 1s”.
- Nguyên nhân: pipeline cũ chưa đồng bộ hoàn chỉnh sang `%`.

- Hiện tượng app đơ khi vào game, bảng điều khiển lag/khó bấm.
- Nguyên nhân chính nghi ngờ cao: burst tick `popup-frame` + log/perf message quá dày + nhiều `Dispatcher.BeginInvoke` liên tục, làm nghẽn UI thread.
- Log xác nhận: có thời điểm hàng chục dòng `[PERF][MSG] abx=tick | src=popup-frame` trong cùng 1 giây.

- Vùng login từng không tự ẩn sau login thành công.
- Nguyên nhân: `SetModeUi` có giai đoạn bị hardcode `GroupLoginNav.Visibility = Visible`.
