# PROJECT CONTEXT

Hệ thống ERP nội bộ cho doanh nghiệp thang máy khoảng 100 người. Trục nghiệp vụ chính:

Khách hàng → Hồ sơ tư vấn → Lịch chăm sóc → Báo giá → Hợp đồng → Dự án → Hồ sơ thang máy → Thi công → KCS → Kiểm định → Nghiệm thu → Bàn giao → Bảo hành → Bảo trì.

Không sử dụng khái niệm “Cơ hội bán hàng”. `Khách hàng` là dữ liệu master định danh cá nhân/doanh nghiệp. `Hồ sơ tư vấn` là đơn vị nghiệp vụ kinh doanh đại diện cho một nhu cầu thang máy/công trình cụ thể, dùng để theo dõi tư vấn, kỹ thuật sơ khai, tài liệu khảo sát, báo giá, hợp đồng và KPI kinh doanh. Dự án chỉ được tạo sau khi hợp đồng đã ký hoặc được phê duyệt triển khai.
